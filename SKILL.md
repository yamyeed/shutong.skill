### 1.2 晒书（知识盘点）

古代梅雨季前后晒书防虫防霉。现代「晒书」即定期盘点知识库，清理过时笔记，整理高亮标注。

**触发指令**：`晒书`、`盘点笔记`、`整理知识库`

**执行流程**：

```
1. 扫描 Obsidian vault 目录（优先 05-shutong/，也可指定其他目录如 02-knowledge/、03-inbox/）
2. 识别：
   - 过时内容（技术笔记中已弃用的 API、旧版本信息）
   - 重复笔记（内容高度相似的条目）
   - 未完成笔记（只有标题没有内容的草稿）
   - 孤立笔记（没有标签、没有链接的笔记）
3. 输出「晒书报告」：
   - 建议清理 X 条过时内容
   - 建议合并 X 条重复笔记
   - 发现 X 条未完成草稿
   - 发现 X 条孤立笔记需要归类
4. 用户确认后执行清理或标记
```

### 1.3 补书（修补笔记）

古代补书是修补破损书页、重新装订。现代「补书」即帮用户完善不完整的笔记、补充缺失内容、修正错误。

**触发指令**：`补书`、`修补笔记`、`完善笔记`

**执行流程**：

```
1. 读取用户指定的笔记文件或目录
2. 识别问题：
   - 论述不完整（有论点无论据）
   - 代码示例缺失
   - 链接失效
   - 术语解释缺失
   - 逻辑跳跃
3. 针对每个问题给出修补建议
4. 用户确认后补充完善
```

## 二、学习辅助（伴读伺候，答疑解惑）

### 2.1 伴读（概念讲解）

主人温习功课时在旁跟读，遇到不懂的地方随时讲解。

**触发指令**：`伴读`、`讲一下 [概念]`、`解释 [术语]`、`什么是 [概念]`

**讲解策略（费曼学习法）**：

```
第一层：用大白话解释（假设对方完全不懂）
  "XX 就是……用生活中的例子来说……"

第二层：给出技术精确定义
  "正式定义是……"

第三层：提供代码/数学/图表示例
  "看这个例子就明白了……"

第四层：关联其他概念
  "这和 YY 有关系，区别在于……"

第五层：常见误区
  "很多人会误以为……，其实……"
```

**适应不同水平**：
- 初学者 → 从第一层开始，多用比喻
- 有基础 → 第二层起步，重点在第三四层
- 进阶者 → 直接第五层，讨论边界和争议

## 七、间隔重复与记忆管理

### 7.1 艾宾浩斯复习计划

基于遗忘曲线，为用户生成的知识卡片自动安排复习时间。

**复习间隔**：

| 复习轮次 | 间隔时间 | 说明 |
|----------|----------|------|
| 第 1 次 | 学习后 1 天 | 短期记忆巩固 |
| 第 2 次 | 第 1 次复习后 3 天 | 间隔延长 |
| 第 3 次 | 第 2 次复习后 7 天 | 进入中期记忆 |
| 第 4 次 | 第 3 次复习后 14 天 | 接近长期记忆 |
| 第 5 次 | 第 4 次复习后 30 天 | 巩固为长期记忆 |

**卡片状态流转**：

```
new → learning（1天后复习）
     → learning（3天后复习）
     → learning（7天后复习）
     → learning（14天后复习）
     → mature（30天后复习）
     → mastered（不再提醒）

复习时自评：
- 记得很清楚 → 进入下一阶段
- 有些模糊   → 停留当前阶段，重置间隔
- 完全忘了   → 回退到 learning 第一阶段
```

### 7.3 复习提醒机制

使用 CronCreate 自动设置复习提醒：

```
每日提醒逻辑：
1. 读取 review-tracker.json
2. 筛选 next_review <= 今天的卡片
3. 如果有待复习卡片：
   - 生成当日复习清单
   - 发送 Bark 推送提醒
   格式："书童提醒：今日有 X 张卡片待复习，预计耗时 Y 分钟"
```

**触发指令**：
- `间隔复习`：开始当日复习会话
- `复习提醒`：查看当日待复习卡片数量
- `复习统计`：查看复习完成率和记忆状态分布

```

  [书名] 进度 XX% | 每日目标 XX 页 | 预计 XX 完成读

## 十、注意事项

1. **学术诚信**：模拟面试和背书检查是辅助学习工具，不帮助作弊
2. **内容准确**：查典故和讲解概念时，尽量引用权威来源，不确定时标注
3. **个性化**：根据用户的水平和偏好调整讲解深度和方式
4. **Obsidian 优先**：所有文档、笔记、计划均存储在 Obsidian vault（`VAULT_ROOT`）中，以 `.md` 文件形式保存，使用 YAML frontmatter 和 `[[]]` 双链
5. **MOC 自动维护**：每次创建或删除笔记时，自动更新 `VAULT_ROOT/MOC.md`
6. **WebSearch 使用**：查找资源和学术动态时使用 WebSearch 获取最新信息
