# AI 书童 (Shutong)

> 古代书童辅佐公子读书科考，今有 AI 书童伴你求知精进。

一个运行在 [Claude Code](https://docs.anthropic.com/en/docs/claude-code) 上的学习陪伴 Skill，将 AI 助手变成你的私人书童——管理书架、伴读答疑、模拟考试、代笔文书、收集情报，全部数据存储在本地 Obsidian 中。

## 核心理念

用古代书童的五大职能映射现代学习场景：

| 古代职责 | 现代功能 | 触发词 |
|---------|---------|--------|
| 整理书架、晒书防虫 | 书籍管理、知识库盘点 | `整理书架` `晒书` `补书` |
| 伴读跟学、背书检查 | 概念讲解、抽检测试 | `伴读` `考我` `背书` |
| 准备考篮、陪同赴考 | 复习计划、考前冲刺、考后复盘 | `备考` `考前冲刺` `考后复盘` |
| 代写书信、誊抄文稿 | 邮件起草、文章润色 | `代写` `润色` `校对` |
| 外出买书、打探消息 | 资源推荐、学术动态追踪 | `找书` `找资料` `学术动态` |

## 功能一览

### 书籍管理（经史子集）

- **整理书架** — 按经（理论）、史（历史）、子（专业）、集（文集）四部分类
- **抄书** — 从 URL/文件/粘贴文本中萃取精华，生成结构化读书笔记
  - `--速读` 一句话+三要点（快速筛选）
  - `--精读` 完整笔记模板（深度学习）
  - `--卡片` Anki 兼容知识卡片（间隔重复）
  - `--脑图` Markdown 思维导图（结构概览）
- **晒书** — 盘点知识库，清理过时/重复/孤立笔记
- **补书** — 修补不完整笔记，补充缺失内容

### 学习辅助

- **伴读** — 费曼学习法五层讲解（大白话→精确定义→示例→关联→误区）
- **考我** — 四种测试模式（填空/问答/概念辨析/代码默写）+ 难度自适应
- **模拟面试** — 支持 2-3 轮追问的技术/产品/学术模拟面试
- **查典故** — 技术概念溯源、论文出处查找
  - `--中文` 成语出处、古文名句、诗词典故

### 考试备考

- **备考** — 生成四阶段复习计划（基础→专项→模拟→冲刺）
- **考前冲刺** — 速记清单 + 心理疏导
- **考后复盘** — 知识盲区分析 + 改进计划

### 文书工作

- **代写** — 邮件/消息/邀请函/感谢信，支持正式/半正式/轻松/古风风格
- **润色校对** — 语法检查 + 用词优化 + 结构调整，输出修改对比

### 信息收集

- **找书** — WebSearch 驱动的分级资源推荐（入门/进阶/实战/免费）
- **找资料** — 论文、开源项目、官方文档精准搜索
- **学术动态** — 技术前沿周报，按领域聚合

### 记忆管理

- **间隔复习** — 艾宾浩斯遗忘曲线，5 级复习间隔（1d→3d→7d→14d→30d）
- **卡片状态流转** — new → learning → mature → mastered
- **每日复习提醒** — CronCreate 自动推送待复习卡片
- **连续复习天数追踪** — streak 机制激励持续学习

### 数据看板

- **书房概览** — 聚合阅读量/知识卡片/复习完成率/备考进度/连续复习天数

### 分享导出

- **导出笔记** — Markdown / 精简书评 / 纯文本 / PDF
- **分享书单** — 生成可分享的阅读清单

## Obsidian 集成

所有数据存储在 Obsidian 中，可在 Obsidian 中直接查看和编辑：

```
shutong/
├── books/              # 读书笔记（每本书一个 .md 文件）
├── notes/              # 知识卡片、概念笔记、复习追踪
├── exam-prep/          # 备考计划、复习资料、速记卡
├── drafts/             # 代写文书、润色草稿
├── reading-plan/       # 阅读计划、进度追踪、书架索引
└── MOC.md              # Map of Content（自动维护的内容索引）
```

笔记格式使用 YAML frontmatter，完全兼容 Obsidian 属性面板。支持 `[[]]` 双链和 `#tag` 标签。

## 安装

将 `shutong` 目录复制到 Claude Code 的 skills 目录：

```bash
cp -r shutong/ ~/.claude/skills/shutong/
```

确保 Obsidian Vault 路径在 `SKILL.md` 中正确配置（默认 `05-shutong/`）。

## 使用示例

```
用户：抄书 https://example.com/article-about-rag
书童：[获取文章] → [生成读书笔记] → [保存到 books/] → [更新 MOC.md]

用户：考我 分布式系统
书童：[从已有笔记提取知识点] → [按难度出题] → [评估回答] → [记录薄弱点]

用户：模拟面试 后端开发
书童：[开始面试] → [自我介绍] → [追问2-3轮] → [综合评价]

用户：考前冲刺
书童：[输出速记清单] → [高频考点] → [易混淆概念对比表] → [心理疏导]

用户：书房概览
书童：[聚合所有数据] → [输出统计仪表盘]
```

## 技术细节

- **运行环境**：Claude Code CLI
- **数据存储**：本地 Obsidian（Markdown + JSON）

- **笔记格式**：YAML frontmatter + Markdown + Obsidian 双链
- **语言**：中文为主
